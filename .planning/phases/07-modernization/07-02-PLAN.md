<task type="auto">
  <name>Task 1: Install PHPStan with WordPress stubs and create configuration</name>
  <files>composer.json, phpstan.neon</files>
  <action>
1. Add PHPStan and WordPress stubs to `require-dev` in `composer.json`:
   - `"phpstan/phpstan": "^2"` (latest stable 2.x)
   - `"szepeviktor/phpstan-wordpress": "^2"` (WordPress-specific rules and stubs)

   Keep ALL existing require-dev entries. If any of the WPCS 3 dependencies are missing on the working branch, add them too: `"squizlabs/php_codesniffer": "^3.10"`, `"wp-coding-standards/wpcs": "^3"`, `"dealerdirect/phpcodesniffer-composer-installer": "^1"`.

   Do NOT change `vendor-dir`, `autoload`, `require`, or any other non-dev settings.

2. Create `phpstan.neon` at the repo root:
```neon
includes:
    - inc/lib/szepeviktor/phpstan-wordpress/extension.neon

parameters:
    level: 5
    paths:
        - comment-popularity.php
        - inc/
        - admin/
    excludePaths:
        - inc/lib/
        - inc/templates/
    bootstrapFiles:
        - comment-popularity.php
    scanDirectories:
        - inc/lib/twig/twig/src
```

   Note: The vendor-dir is `inc/lib`, so the extension path uses `inc/lib/` not `vendor/`. The `scanDirectories` entry lets PHPStan resolve Twig classes without including them in the analysis.

3. Run `composer install --no-interaction` to install the new deps, then run `inc/lib/bin/phpstan analyse --generate-baseline` to create `phpstan-baseline.neon`. Add the baseline include to `phpstan.neon`:
```neon
includes:
    - phpstan-baseline.neon
    - inc/lib/szepeviktor/phpstan-wordpress/extension.neon
```
